Git commands

* Use in Terminal *

  • git init
    •

    Initializes a Git repository in a local directory.

    Example: git init

  • git clone
    •

    Clones an existing Git repository to your local directory.

    Example: git clone "repository_url"

  • git add
    •

    Adds files to the index (staging area) to prepare them for commit.

    Example: git add "file.txt"

  • git commit
    •

    Records changes to the repository. You need to provide a commit message to describe the changes.

    Example: git commit -m "Your commit message here."

  • git push
    •

    Sends local changes to a remote repository.

    Example: git push origin main

  • git pull
    •

    Updates the local repository with changes from the remote repository.

    Example: git pull origin main

  • git status
    •

    Displays the current state of the working directory and the index (staging area).

    Example: git status

  • git branch
    •

    Lists all branches in the repository. The asterisk (*) indicates the current branch.

    Example: git branch

  • git checkout
    •

    Switches to a specific branch or restores files from the working directory.

    Example (switch to a branch): git checkout "branch_name"

    Example (restore a file): git checkout --"file.txt"

  • git merge
    •

    Combines the history of another branch into the current branch.

    Example: git merge "branch_name"

  • git remote
    •

    Displays and manages remote repositories and their connections.

    Example: git remote -v

  • git log
    •

    Displays the commit history of the repository.

    Example: git log

  • git reset
    •

    Undoes changes in the index (staging area) and/or the working directory.

    Example (go back to the last commit and keep changes in the working directory): git reset --soft "HEAD˜1"

    Example (go back to the last commit and discard changes in the working directory): git reset --hard "HEAD˜1"
