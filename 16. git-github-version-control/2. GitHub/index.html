GitHub commands

* Use in Terminal *

  • git clone
    •

    This command allows you to clone a repository from GitHub to your local machine.

    Example: git clone "repository_url"

  • git add
    •

    This command stages changes in your working directory for the next commit. You can add specific files or use a period to add all changes.

    Example: git add "file.txt"

  • git commit
    •

    This command pushes your committed changes to the remote repository on GitHub.

    Example: git commit -m "Your commit message here."

  • git push
    •

    This command pushes your committed changes to the remote repository on GitHub.

    Example: git push origin main

  • git pull
    •

    This command fetches and merges changes from the remote repository to your local repository.

    Example: git pull origin main

  • git status
    •

    This command shows the current status of your repository, including the modified files and staged changes.

    Example: git status

  • git branch
    •

    This command shows the list of branches in your repository. An asterisk (*) indicates the currently checked-out branch.

    Example: git branch

  • git checkout
    •

    This command allows you to switch between branches or restore files from a specific commit.

    Example: git checkout "branch_name"

  • git merge
    •

    This command merges a branch into the current branch.

    Example: git merge "branch_name"

  • git remote
    •

    This command shows the list of remote repositories associated with your local repository.

    Example: git remote -v

  • git log
    •

    Displays the commit history of the repository.

    Example: git log
